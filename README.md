# UI-UX-Experiment5

## Aim:
To design and develop user-friendly web applications for a Medicine Delivery System (Pharm-Care) and an E-commerce Platform by applying UX design principles, enabling users to easily place orders and receive confirmation through a simple and efficient interface

## ALGORITHM

1.Identify user problems in online ordering systems.

2.Define clear UX problem statements based on user needs.

3.Select appropriate research methods such as surveys, interviews, and usability testing.

4.Analyze user behavior and create user personas.

5.Design simple and intuitive user interfaces for both applications.

6.Create input forms to collect user details (name, phone, address).

7.Provide options for selecting medicines/products and quantity.

8.Implement a “Place Order” functionality.

9.Store user input data using local storage.

10.Redirect users to a confirmation page after order placement.

11.Display the order details clearly on the confirmation page.

12.Ensure proper navigation, usability, and consistency in design.

## Output:
🌐 💊 Pharm-Care (Medicine Delivery App)
🔹 1. Problem Statement

* Users find it difficult to quickly order medicines online

* Lack of simple interfaces causes confusion

✅ UX Problem Statements:

* “Users need a way to order medicines quickly because complex interfaces delay the process, which results in frustration.”

* “Elderly users need a simple medicine ordering system because they are not familiar with technology, which results in missed or delayed purchases.”

🔹 2. Research Methods
Methods Used:

* Surveys

* User Interviews

* Usability Testing

Justification:

* Surveys help understand common user needs

* Interviews identify real-life problems (especially elderly users)

* Usability Testing ensures the app is easy to use

👉 UX design process involves research, personas, and testing to improve usability

🔹 3. User Personas
👤 Persona 1: Ramesh

* Age: 60

* Occupation: Retired

* Goal: Order medicines easily

* Behavior: Uses phone occasionally

* Pain Points: Confusing apps

* Motivation: Convenience & health

👤 Persona 2: Priya

* Age: 30

* Occupation: Working Professional

* Goal: Quick medicine delivery

* Behavior: Uses apps daily

* Pain Points: Slow checkout

* Motivation: Save time

🔹 4. Solution (Design)

* Simple order form

* Medicine dropdown selection

* Quantity input

* Address input

* “Place Order” button

* Confirmation page after order

🔹 5. Result

* A user-friendly medicine delivery web application (Pharm-Care) was designed where:

* Users can order medicines easily

* Data is stored temporarily

* Confirmation page improves user trust

🛒 🌐 E-Commerce Website
🔹 1. Problem Statement

* Users struggle to find products quickly

* Checkout process is complicated

✅ UX Problem Statements:

* “Users need a way to quickly find products because too many options create confusion, which results in frustration.”

* “Users need a simple checkout process because complex steps reduce trust, which results in cart abandonment.”

🔹 2. Research Methods
Methods Used:

* Usability Testing

* Surveys

* A/B Testing

Justification:

* Usability Testing identifies navigation problems

* Surveys collect user feedback

* A/B Testing improves UI decisions

🔹 3. User Personas
👤 Persona 1: Ravi

* Age: 35

* Occupation: Business Owner

* Goal: Fast shopping

* Behavior: Frequent online buyer

* Pain Points: Slow navigation

* Motivation: Save time

👤 Persona 2: Anitha

* Age: 27

* Occupation: Teacher

* Goal: Safe purchase

* Behavior: First-time buyer

* Pain Points: Trust issues

* Motivation: Secure shopping

🔹 4. Solution (Design)

* Product selection dropdown

* Quantity input

* Address input

* Simple checkout button

* Order confirmation page

🔹 5. Result

* A simple e-commerce web application was designed where:

* Users can easily place orders

* Checkout is simplified

* Confirmation page improves experience
